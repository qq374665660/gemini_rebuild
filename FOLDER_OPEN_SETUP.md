# 项目文件夹直接打开功能设置指南

## 功能说明

"打开项目文件夹"按钮现在可以直接打开Windows文件资源管理器到指定的项目目录，无需手动复制路径。

## 设置步骤

### 1. 注册自定义协议处理器

为了让浏览器能够直接调用Windows文件资源管理器，需要注册一个自定义URL协议。

**方法一：使用注册表文件（推荐）**

1. 以管理员身份运行命令提示符或PowerShell
2. 导航到项目目录：`cd G:\gemini_rebuild`
3. 双击运行 `setup_protocol_handler.reg` 文件
4. 在弹出的确认对话框中点击"是"

**方法二：手动执行批处理文件**

如果注册表方法不工作，可以直接测试批处理文件：

```bash
# 测试协议处理器
.\register_url_protocol.bat "cscec-pm://open-folder/2024-在研-TEST001-测试项目"
```

### 2. 验证设置

1. 打开项目管理系统：http://127.0.0.1:10086/
2. 进入任意项目详情页面
3. 点击"打开项目文件夹"按钮
4. 如果设置成功，Windows文件资源管理器应该会直接打开到项目目录

### 3. 故障排除

**如果按钮不工作：**

1. **检查协议注册**：
   - 确保以管理员权限运行了注册表文件
   - 检查注册表中是否存在 `HKEY_CLASSES_ROOT\cscec-pm` 项

2. **检查路径设置**：
   - 确保 `register_url_protocol.bat` 中的 `PROJECTS_BASE_PATH` 设置正确
   - 默认路径：`G:\gemini_rebuild\projects`

3. **浏览器安全设置**：
   - 某些浏览器可能会阻止自定义协议
   - 尝试在浏览器中允许自定义协议的执行

4. **备用方案**：
   - 如果自定义协议不工作，系统会自动回退到显示文件夹路径的模态框
   - 用户可以复制路径并手动在文件管理器中打开

## 技术实现

### 工作原理

1. **自定义协议**：`cscec-pm://open-folder/{项目路径}`
2. **协议处理器**：`register_url_protocol.bat`
3. **回退机制**：如果自定义协议失败，尝试 `file://` 协议，最后显示路径复制界面

### 文件说明

- `setup_protocol_handler.reg`：Windows注册表文件，注册自定义协议
- `register_url_protocol.bat`：协议处理器，接收URL并调用Windows资源管理器
- `project_detail.html`：包含修改后的 `openProjectFolder()` JavaScript函数

### 安全考虑

- 协议处理器只能打开项目根目录下的文件夹
- 路径验证确保不会访问系统敏感目录
- 如果目标文件夹不存在，会显示错误信息

## 支持的操作系统

- ✅ Windows 10/11
- ❌ macOS（需要不同的实现）
- ❌ Linux（需要不同的实现）
